# GitHub Issues and Pull Requests

In this session we learn how to contribute to repositories through **Issues** and how to suggest changes via **Pull Requests** — both leaving a permanent discussion record in your repository.

---

## Step 1: File an issue on your own repository

Go to your recipe repository and click the **Issues** tab (next to Code).

Issues are great for tracking ideas you intend to implement. Click **"New issue"**:

<img src="/img/contributing/new_issue.jpg" class="border rounded mt-2 max-h-64 mx-auto" />

Add a title and description of an improvement for your recipe repository.

> You can assign issues to specific people, and categorize them with labels and milestones.

---

## Step 2: File an issue on a colleague's repository

Issues are commonly used to report bugs or request features in someone else's project.

<div class="p-4 border border-green-400 rounded bg-green-50">

**Exercise**

- In the workshop EtherPad, write your name and a link to your GitHub repository.
- Once everyone has added their link, go to the repository of the person **above you** and file a bug report or feature request as a new Issue.
- How were you alerted when someone filed an issue in your repository?
- Find the issue filed in your repository and respond to it.

</div>

---

## Step 3: Make a pull request

Anyone can suggest changes to an open source project. Navigate to someone else's repo and click the **pencil icon** to edit a file.

Notice the difference — instead of "Commit changes", you see:

<img src="/img/contributing/fork_top.jpg" class="border rounded mt-2 max-h-44 mx-auto" />

---

At the bottom there is a **"Propose changes"** button instead of "Commit changes":

<img src="/img/contributing/fork_bottom.jpg" class="border rounded mt-2 max-h-44 mx-auto" />

Clicking it creates a **fork** of the repository under your account and opens a comparison view.

---

The **"Compare Changes"** page shows exactly what you changed:

<img src="/img/contributing/compare_changes.jpg" class="border rounded mt-2 max-h-60 mx-auto" />

Click the green **"Create Pull Request"** button to submit your suggestion.

> This is only a *request* — the owner can accept, reject, or discuss your changes.

---

Go back to your own repository and check if a Pull Request has come in from a classmate. Respond to it however you feel is appropriate.

<div class="p-4 border border-blue-400 rounded bg-blue-50 mt-4">

**Key idea:** Pull Requests make open-source collaboration possible — anyone can propose a change, and the project maintainer stays in control.

</div>
